Architecture

Five layers, one ingest.

A single interaction is consolidated across every relevant layer; a single query assembles a unified, layer-aware <memory_context>.

IWORKING

Working memory RAM

A volatile ring buffer of the last N turns — fast, recency-ordered context for the current session.

IIEPISODIC

Episodic memory SQLite

The durable diary: every raw user/assistant turn, reconstructable verbatim, plus narrative day summaries.

IIISEMANTIC

Semantic memory sqlite-vec

Decontextualized fact vectors — extracted, deduplicated, and retrieved by cosine similarity with multi-signal scoring.

IVPROCEDURAL

Procedural memory rules

Standing behavioral directives — “Respond in German.”, “Keep answers concise.” — injected so the model's behavior persists.

VDIARY

Hierarchical diary optional

A bounded, LLM-written narrative — a rolling per-session entry folded into one growing persistent summary, via a provider-agnostic outbox.

Inter-layer transfer

The glue that moves information between layers so the context window stays small and salient.

01
Consolidation

One ingest fans out — working buffer, semantic facts, episodic log, and procedural rule detection.

02
Retrieval

One query assembles the unified <memory_context>: directives + scored facts + recent dialogue.

03
Distillation

Recurring high-importance preferences graduate from facts into standing procedural directives.
