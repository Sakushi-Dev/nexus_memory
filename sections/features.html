Why Nexus

A memory you can drop in — and reason about.

Everything goes through one validated entry point, runs on a single local file, and never hides what it's doing.

⌘

One entry point

Send a dict (or JSON) to process(), get a dict back. It validates every request with pydantic and never raises to the caller — treat memory as a black box that always answers.

⬡

Local & offline

A single .db file (SQLite + sqlite-vec). The default embedder is a deterministic, dependency-free hashing vectorizer — no server, no network, no model download.

▤

5-layer cognitive memory

Working, episodic, semantic and procedural memory — modeled on Atkinson–Shiffrin — plus an optional hierarchical diary. One ingest fans out across all of them.

⇄

Provider-agnostic

The module never calls an LLM. When a summary is due it enqueues a job into a handoff outbox; your host runs it on any model and hands the text back. Nexus owns the prompt, you own the model.

◎

Transparent & sovereign

Inspect, pin, update and forget your own memories. A live trace surfaces the module's internal steps — the real 4-layer write fan-out and retrieval, nothing fabricated.

⌖

Salient retrieval

vec0 cosine KNN plus multi-signal scoring (similarity × importance × decay) keep the working context small and on-point — mitigating the lost-in-the-middle problem.
