FAQ

Questions, answered.

The short version — local-first, provider-agnostic, and yours to inspect.

What is Nexus Memory?
A local-first cognitive memory for LLM agents. Instead of a flat chat log, it gives your agent working, episodic, semantic and procedural memory behind a single process() call — so it actually remembers facts, events and your standing preferences across sessions. It runs as one local .db file, offline and provider-agnostic. See Features, the Architecture, or read the Overview in the docs.
How do I install it?
Two steps: create a project-local virtualenv and do an editable install — python -m venv .venv, then pip install -e .. There's no API key or model download for the default offline path. The full copy-paste snippet (plus the offline self-test) is in the Quickstart, and a deeper walkthrough lives in the docs.
How do I use it once it's installed?
Everything goes through one entry point: process(). You ingest an interaction to store it and assemble a context (plus standing directives) to read it back — then inject those into your system prompt. The Quickstart shows a working example; the I/O map and I/O docs list every action and its inputs and outputs.
Does Nexus need an API key or internet?
No. The default path is fully offline: a single local .db file (SQLite + sqlite-vec) and a deterministic, dependency-free hashing embedder — no server, no model download. Want semantic recall? Drop in the optional local embedder (fastembed + a small open model): it downloads once, then runs offline on CPU — so you stay vendor-free. An LLM for the diary / procedural side tasks is also optional, on any provider you choose.
Can it recall a fact when I ask in different words?
Yes — with the optional local semantic embedder (0.7.0). The zero-dependency default matches on shared words, so it can miss paraphrases. Switch on the local fastembed embedder (pip install "nexus-memory[local-embeddings]") and "what's my project's name?" retrieves "I'm building a tool called Tideglass" with no words in common — all on-device, no API. Switching is a one-command re-index.
How is it different from a vector database?
A vector DB stores embeddings; Nexus is a cognitive memory. It layers working, episodic, semantic and procedural memory behind one process() call, and moves information between them — consolidation on write, a unified context on read, and distillation of recurring preferences into standing behavior.
Which LLM providers are supported?
All of them — because the module never calls an LLM itself. When a background task is due — a diary summary or procedural rule extraction — it enqueues a job into a handoff outbox; your host runs it on any provider (and you can point those side tasks at a separate, cheap “aux” model while a stronger model handles the chat).
Is my data private?
Yes. Everything lives in one local file you control. There's opt-in regex PII masking before embedding, an off-path SQLCipher encryption hook, and full transparency — inspect, pin, update and forget any memory at will.
Can I use a real embedding model?
Yes. Lazy SentenceTransformer and OpenAI embedder adapters are included and load only when you ask for them. The dependency-free HashingEmbedder is just the default so the library works out of the box, offline.
What is Layer V, the diary?
An optional, off-by-default layer that keeps a bounded narrative of the conversation — a rolling per-session entry folded into one growing persistent summary, written via the outbox. Delete the layers/diary/ folder and Nexus runs exactly as before.
How fast is it — and does it scale?
Retrieval is an indexed vec0 cosine KNN plus a cheap multi-signal score — ~3 ms to assemble a context locally. Writes run on a background pipeline, and WAL mode keeps reads and writes concurrent on the shared connection.
What are the requirements and license?
Python 3.11+, with SQLite + sqlite-vec, pydantic and numpy. It's MIT licensed — drop it in and keep both the model and the data yours.
