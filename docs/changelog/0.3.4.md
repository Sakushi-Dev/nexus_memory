# 0.3.4 — 2026-06-19

An additive release: optional **tiktoken** token counting, a single shared
token-estimation heuristic, a diary reconciliation clause, and an examples
overhaul. Backward compatible — no schema changes.

## Added

- **Optional `tiktoken` backend for `tokens()`** via the new `config=` argument:
  `config="tiktoken"`, a model name (e.g. `"gpt-4o"`), or a
  `{"model"|"encoding": ...}` dict selects exact token counts through the optional
  extra (`pip install nexus-memory[tiktoken]`). The default stays the offline
  `len(s) // 4` heuristic. Resolved by `xml_format.resolve_counter`.
- **`examples/token_accounting.py`** — a focused tour of every `tokens()` option
  (scopes, list+total, `config=` counting methods, prompt-vs-completion, edge cases).

## Changed

- **`tokens(config=...)`** replaces the earlier `counter=` parameter: `config`
  accepts `None` (default heuristic), a `(str) -> int` callable, a
  `"tiktoken"`/model-name string, or a `{model|encoding}` dict.
- **Single source of truth for token estimation.** `history(max_tokens=)` and
  `WorkingMemory.token_estimate()` now delegate to the shared
  `xml_format.estimate_tokens` (`len(s) // 4`) instead of re-implementing it.
- **Diary `DAILY_PROMPT` reconciliation clause** — when a newer turn corrects or
  contradicts the prior entry, the model treats the newer turn as authoritative
  and revises the earlier wording rather than keeping both.
- **Examples overhauled.** `basic_usage.py` is now a minimal OpenAI-format
  central-memory loop (system + `history()` + user turn); `diary_outbox.py` shows
  the rolling diary feeding `prior_summary` back across two rounds with a
  reconciling stub. Both remove their demo DB on exit.

## Documentation

- `tokens()` API reference updated for `config=` and the optional tiktoken backend.
