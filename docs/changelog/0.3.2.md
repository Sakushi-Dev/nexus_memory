# 0.3.2 ‚Äî 2026-06-19

An additive release centered on a new **unified history accessor** that turns the
durable episodic layer into a native LLM message history, plus a robustness fix
for the diary drain and a setup-docs update. Fully backward compatible ‚Äî no
breaking changes to existing APIs or stored data.

## Added

- **`NexusMemory.history(...)`** ‚Äî a method-only convenience accessor that returns
  the conversation history ready to feed straight into a chat LLM. It reads the
  **durable episodic store** (Layer II) ‚Äî or, when `episodic_enabled=False`, the
  volatile working buffer (Layer I) ‚Äî mirroring the same source selection as
  `assemble`'s `recent_dialogue`, so the history **survives a restart**. No new
  tables: it is a read-only view over storage Layers II/I already own. Features:
  - **Three output formats** via `as_format`: `"messages"` ‚Üí `[{role, content}]`
    (drop-in chat history, default), `"turns"` ‚Üí adds `timestamp`, `"string"` ‚Üí
    a newline-joined transcript rendered via `template`.
  - **Two truncation modes**: by turns (`max_turns`) or by token budget
    (`max_tokens` with an optional `token_counter`, defaulting to the
    `len(s) // 4` heuristic). Explicit args win over config defaults, and
    `max_tokens` takes precedence over `max_turns`.
  - **Role filter** (`role="user"`/`"assistant"`), applied before truncation.
  - Turns are always chronological (newest-last). Invalid `as_format`/`role`
    raise `ValueError`.

  See [history()](../usage/api-reference.md#history--native-message-history) and
  [Memory Layers](../architecture/memory-layers.md#unified-history-over-working--episodic).
- **`NexusConfig` history knobs**: `history_truncation` (`"turns"`/`"tokens"`,
  default `"turns"`), `history_max_turns` (default `20`), and
  `history_token_budget` (default `2000`). `history_truncation` is validated at
  construction.

## Changed

- **`drain_diary` surfaces silent host failures.** When the diary is enabled and a
  job is pending but `run_job` returns no summary (an empty result, or a host that
  swallowed a model error), the job is now skipped **and a `WARNING` is logged** ‚Äî
  so a silently broken host model surfaces instead of leaving the diary entry
  blank.
- **`basic_usage.py` rewritten** to demonstrate the recommended split when Nexus
  is your central memory: `assemble` feeds the **system prompt** (Layer III/IV
  directives + recalled facts), while `memory.history()` provides the **native
  message array** (Layer II/I conversation) ‚Äî avoiding a duplicated
  `<recent_dialogue>` block and keeping roles native.

## Documentation

- **Setup switched to `pip install -e .`** (from the previous clone-and-embed
  model), with virtualenv and update instructions.
- **New `history()` reference** in the API docs and a **"Unified history over
  Working / Episodic"** section in the memory-layers architecture page.
- Fixed a broken API-reference anchor in `request-response.md`.
