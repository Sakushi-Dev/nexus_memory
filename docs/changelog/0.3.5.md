# 0.3.5 — 2026-06-19

A diary (Layer V) rework: the daily entry is now the **assistant's own
first-person journal** in flowing prose, sized to how much happened, fed by a
**rolling, overlapping window** of recent turns. No schema change — existing
diaries are unaffected. One behavior change: the default cadence moved from
`update_every=3` to `update_every=5` (pin `update_every=3` to keep the old rate).

## Added

- **`DiaryConfig.diary_window`** (default `20`) — the number of **turns**
  (1 turn = a user message + the assistant reply = 2 rows) re-sent in each
  rolling daily job. The job now carries an **overlapping** window — at least the
  last `diary_window` turns of the day (for reconciliation) and everything
  ingested since the last applied drain (completeness) — instead of a strict
  "new turns since `covered_through`" delta. Distinct from
  `NexusConfig.history_max_turns`, which caps the chat-history accessor.
- **`DiaryConfig.max_sentences`** (default `50`) — the upper bound of the daily
  entry's `2-max_sentences` sentence range. `DAILY_PROMPT` is now a template with
  a `{max_sentences}` placeholder, filled at enqueue time. The old fixed 2-5 cap
  throttled the whole-day entry and could never grow as a day re-summarized; the
  floor stays 2.
- **`DiaryConfig.__post_init__` validation** (runs regardless of `enabled`):
  `update_every`, `diary_window`, `section_size`, `max_sections` must be `>= 1`;
  `max_sentences >= 2`; `inject_days >= 0`. Out-of-range values raise
  `ValueError`.

## Changed

- **Default daily cadence `update_every` 3 → 5.** Existing `diary=True` hosts
  enqueue ~40% fewer daily jobs. **Migration:** pin
  `diary=DiaryConfig(enabled=True, update_every=3)` to keep the prior cadence.
- **`DAILY_PROMPT` rewritten** to the assistant's **first-person voice** ("I"),
  reflecting the whole exchange (both the user's and the assistant's turns), as
  **flowing prose** (never bullet points, numbered lists, headings, or
  categorical structure), `2-{max_sentences}` sentences sized to how much
  happened. It also now names the overlap: the recent turns may include turns
  already in the prior entry, so the model incorporates only genuinely new
  developments rather than restating them. The reconciliation clause
  (newer turn wins on contradiction) is retained.
- **`SECTION_PROMPT` rewritten** to the same first-person voice — weaving the
  prior section summary and a new day's entry into a single coherent paragraph of
  flowing prose.
- **The daily window is overlapping, not a delta.** `covered_through` no longer
  *gates* the window; it is now a monotonic **last-applied high-water mark** plus
  the idempotency signal for a new **empty-tick guard** (skip a redundant job
  when nothing advanced since the last apply). Day finalize/rollover paths
  bypass the guard (`force=True`) so a finalized-but-unfolded day is never
  stranded. Overlap-correctness is now a model-quality dependency, mitigated by
  the prompt and the example stub.
- **`examples/diary_outbox.py`** updated for the new cadence (5 interactions per
  round) and a both-role, first-person, **reconciling** stub model (it keeps the
  prior entry and weaves in only genuinely new developments rather than naively
  appending the overlap).

## Documentation

- Diary config, architecture, and use-case pages updated for the first-person
  prose voice, the `2-N` sentence range, the rolling overlapping window, and the
  `diary_window` / `max_sentences` knobs; cadence references bumped 3 → 5.
