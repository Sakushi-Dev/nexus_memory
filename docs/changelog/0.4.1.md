# Unreleased (0.4.1)

A correctness-and-honesty pass driven by a full project audit. Several
documented-but-inert request fields are now either implemented or removed so the
public contract matches the code, a silent read-after-write staleness bug is
fixed, and two destructive/lossy edge cases (unrelated `forget`, cross-namespace
dedup) are closed. No diary-layer schema change; the session model shipped in
0.4.0 is unchanged.

## Added

- **`pin` and `update` are real `process()` actions** (and convenience wrappers
  on `NexusMemory`). `pin` → `{action:"pin", content, importance=10.0}` /
  `memory.pin(content, importance=10.0)` stores a high-importance fact;
  `update` → `{action:"update", target_id, new_content}` /
  `memory.update(target_id, new_content)` replaces a fact's content and re-embeds
  it. Previously these existed only on the internal transparency interface, so the
  advertised "inspect, forget, pin, and update your own memories" surface is now
  actually reachable.
- **`NexusConfig.forget_min_similarity`** (default `0.6`) — a relevance floor for
  `forget(query=…)`. The best KNN match is deleted only when its cosine similarity
  (`1 - distance`) is at or above this value; otherwise `forget` returns
  `{status:"not_found"}` (with `best_similarity`) instead of deleting. This closes
  a silent, irreversible data-loss path where an unrelated query deleted the
  nearest row on any non-empty store.

## Changed

- **`ingest.priority` (1–10) now works.** It acts as an **importance floor**:
  every fact extracted from that interaction is stored with at least `priority`
  (clamped to `[1, 10]`), never lowering a higher heuristic importance. It was
  previously validated, documented, then silently discarded.
- **The semantic read cache is invalidated on every mutation.** Each write path
  — async/sync `ingest` (via the writer's completion callback), `forget`, `pin`,
  `update`, `optimize`, rule add/deactivate, and `distill` — now clears the cache,
  so a query issued after a write no longer returns a stale assembled result that
  omits a just-ingested fact (or still shows a deleted one).
- **Writer dedup is namespace-aware.** A textually near-duplicate fact is only
  collapsed when its nearest neighbour shares the candidate's logical scope
  (`namespace` / `tenant` / `user` / `user_id` / `source` metadata). Near-dupes
  carrying a different scope are kept, so cross-namespace writes are no longer
  silently dropped. With no scope metadata the check reduces to the prior
  similarity-only behaviour.
- **Foreground reads are serialized against the writer.** The shared SQLite
  connection is guarded by a re-entrant lock covering both the layer-store writes
  and the reader, so a foreground reader never interleaves a cursor with the
  background writer thread's commit.
- **Persona-name capture drops filler stopwords.** The procedural "call me … /
  address me as …" rule no longer absorbs trailing stopwords (`please`, `just`,
  `bitte`, `mal`, …) into the captured name; honorific titles (`Dr.`, `Mr.`) are
  preserved.

## Removed

- **`assemble.filters`.** The `AssembleRequest.filters` field was accepted and
  passed through but never consumed on the read path, so callers passing
  `filters={…}` silently got unfiltered results. The field is removed; `assemble`
  now accepts only `query`, `top_k` (default `5`), and `min_score` (default
  `0.6`). A scoped-retrieval design is deferred to a later release.

## Documentation

- Removed all claims that retrieval is "graph-expanded" / expands one hop over
  `memory_edges`. Edge creation was never wired on the write path, so the
  expansion contributed nothing; the latent code stays but is no longer an
  advertised feature.
- Documented `pin`/`update` in every action table and the convenience-wrapper
  list; documented the `ingest.priority` floor, the `forget` relevance floor and
  `forget_min_similarity`, and the removal of `assemble.filters`.
- The landing page now reports **0.4.0** as the latest release (was stale at
  `0.3.2`), and `__version__` reads `0.4.0` to match the packaged version.
- Clarified that `<directive priority="N">` in the assembled XML is a synthetic
  list-rank, not the stored 1–10 priority.

## Tests

- The suite is now **216 tests**, covering the cache-invalidation regression
  (ingest → query → ingest-needle → re-query), the `forget` relevance floor, the
  namespace-aware dedup behaviour, and the `pin`/`update` actions and wrappers.
