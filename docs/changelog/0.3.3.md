# 0.3.3 — 2026-06-19

An additive release adding a **unified token accountant** (`NexusMemory.tokens()`)
and an OpenAI-flavored rewrite of the basic example. Fully backward compatible —
no breaking changes to existing APIs or stored data.

## Added

- **`NexusMemory.tokens(...)`** — a method-only convenience accessor that counts
  tokens over the **actual LLM round-trip** (the request `messages` array plus the
  model's `response`) and splits it by **section**, not storage layer:
  - **`system`** — the whole system message(s): the host's base prompt *and*
    everything Nexus injects into it (`directives` + `facts`). Recalled
    facts/directives live inside the system message, so they count here.
  - **`input`** — the rest of the prompt: every `user`/`assistant` message
    (conversation history + the current user turn).
  - **`output`** — the model's `response` (the completion).
  - **`full`** — `system` + `input` + `output` (the default scope).

  A single scope returns an `int`; a list returns a `{scope: int}` dict with a
  summed `"total"` key. The `counter` defaults to the shared `len(s) // 4`
  heuristic (the same one `history(max_tokens=)` uses), and accepts any
  `(str) -> int` callable (e.g. `tiktoken`) for exact counts. Unknown scopes raise
  `ValueError`. It reads nothing from storage — you hand it the array you actually
  sent, so it counts exactly what crossed the wire.

  See [tokens()](../usage/api-reference.md#tokens--token-accounting).

## Changed

- **`basic_usage.py` rewired for the OpenAI Chat Completions API.** The system
  prompt is now the first `{"role": "system", ...}` entry of a single `messages`
  array (OpenAI has no separate `system` parameter), followed by `memory.history()`
  and the current user turn. Each turn prints `memory.tokens(...)` over the real
  request + response, and the demo DB is removed on exit so every run starts clean.

## Documentation

- **New `tokens()` reference** in the API docs, with the section-based scope table
  and a usage snippet.
