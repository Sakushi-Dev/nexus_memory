# 0.3.1 — 2026-06-18

A small, additive release: two ergonomic conveniences for the optional diary
(Layer V), a clone-and-embed setup model, and a documentation overhaul. Fully
backward compatible — no breaking changes to existing APIs or stored data.

## Added

- **`diary=True` shorthand** on the `NexusMemory` constructor. `diary=True` is
  normalized to `DiaryConfig(enabled=True)` (defaults); pass a `DiaryConfig` for
  custom knobs, and `False`/`None` leaves the layer off. See
  [DiaryConfig](../configuration/diary-config.md).
- **`NexusMemory.drain_diary(run_job)`** — a one-call host-side drain for the
  Layer V outbox. It loops `pending_summaries()`, runs the host callable
  `run_job(job: dict) -> str` on each job, and folds every non-empty result back
  via `submit_summary()`. Returns the number of jobs applied (`0` when the diary
  is off). The module still never imports an LLM SDK — `run_job` is the host's
  model. See [API Reference](../usage/api-reference.md) and
  [Diary Layer](../architecture/diary-layer.md).

## Changed

- **Examples simplified.** `basic_usage.py` and `diary_outbox.py` now use the
  default `db_path` (no `tempfile` scaffolding); the diary demo uses
  `drain_diary` and explains the asynchronous, out-of-band, secondary-model
  handoff.
- **Packaging.** Dropped the duplicate root `schema.sql`; the packaged copy under
  `src/nexus_memory/` is loaded directly. Package author set to `Sakushi-Dev`.

## Documentation

- **Setup reworked to clone-and-embed** (no install step): clone the repo and add
  `src/nexus_memory/` to your project; dependencies come from
  `src/requirements.txt`. Optional embedder install hints — and the matching
  `ImportError` messages — now point at the direct dependencies
  (`pip install sentence-transformers` / `pip install openai`).
- **New [Configuration](../usage/configuration.md) page** — a one-stop map of the
  constructor arguments, all `NexusConfig` fields, and `DiaryConfig`; plus a
  "Beyond NexusConfig" section on the [NexusConfig](../configuration/nexus-config.md)
  page.
- **Diary docs** document the `diary=True` shorthand throughout and the new
  `drain_diary` helper in the API reference and the diary-layer architecture page.
- **Docs tree reorganized** into categorized sections; code comments rewritten to
  describe the code itself (build-history and spec references removed).
