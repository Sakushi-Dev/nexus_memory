# 0.7.0 — 2026-06-21

Adds a **provider-agnostic, locally-downloaded semantic embedder** as an opt-in,
while keeping the zero-dependency `HashingEmbedder` the default. The lexical
hashing embedder only matches on **shared words**, so a paraphrased query
("what is the exact name of my project and which language?") could not retrieve a
fact it shares no salient words with ("I'm building a Rust CLI tool called
Tideglass"). The new `fastembed` backend (BAAI/bge-base-en-v1.5, ONNX, torch-free)
downloads a model **once** and then runs fully offline — a real semantic recall
without becoming a vendor/API embedder. See the
[design doc](../design/local-embedder.md).

768-dim by design — equal to the existing DB vector width — so adopting it needs
**no SQL schema change**; migration is a pure re-embed.

## Added

- **`FastEmbedEmbedder`** (`core/embeddings.py`) — lazy-imported `fastembed`
  adapter, single L2-normalize (bge-`*`-en-v1.5 is already normalized), real
  dim probe, configurable cache, and a robust `offline` path (clear error when
  the model is absent and offline is forced). Exported from the package root.
- **Four `NexusConfig` knobs** (not schema changes): `embedder_backend`
  (`"hashing"` | `"fastembed"`, validated), `embedder_model`, `embedder_cache_dir`,
  `embedder_offline`.
- **`pip install "nexus-memory[local-embeddings]"`** extra (`fastembed[cpu]>=0.8`;
  CPU-only so onnxruntime-gpu is never pulled). Core dependencies unchanged.
- **Embedder provenance** in the existing `system_config` table (rows, zero DDL):
  the active backend/model/dim is stamped on a fresh DB and reconciled on open.
  Opening a store with a different embedder than it was built with is **refused**
  with an error pointing at the re-index tool — no silent vector-space mixing.
- **Dim guard** against the REAL stored vector dim (provenance, else `config.dim`)
  on BOTH the config-built and injected (`embedder=`) paths.
- **`python -m nexus_memory.reindex --db <path> --backend fastembed [--model ...]`**
  (`core/reindex.py`) — transactional **same-dim** re-embed: recompute every
  vector in one transaction, then flip the provenance. A different-dim rebuild is
  out of scope for 0.7.0 (raises `NotImplementedError`).

## Changed

- The orchestrator builds the embedder declaratively from the config
  (`_build_embedder`); an explicit `embedder=` still wins. The semantic cache is
  flushed whenever the active embedder is first stamped onto a store.

## Compatibility

- **Default behavior unchanged.** A bare `pip install nexus-memory` and a plain
  `NexusMemory()` are byte-identical to 0.6.0: `HashingEmbedder`, offline, no
  downloads; a default DB's provenance is `hashing`.
- No database migration and no public API removed. Switching an existing hashing
  store to fastembed requires a one-time `python -m nexus_memory.reindex`.

## Verification

- Full suite green (**237 passed**): 6 new tests cover the fastembed adapter
  (dim + unit norm), the headline A/B recall regression (the Tideglass paraphrase
  is retrieved under fastembed, missed/ranked-far-lower under hashing), the
  dim-guard rejection (no corrupt write), provenance-mismatch refusal (names
  reindex), the same-dim re-embed round-trip (count/content preserved, vectors
  changed, paraphrase now recalls), and offline construction after a warm cache.
  The neural tests are guarded with `pytest.importorskip("fastembed")` so the
  core suite stays dependency-free.
