# 0.5.0 — 2026-06-21

A structural refactor with **zero behavior change**: the diary's job outbox is
generalized into a single, layer-agnostic **AuxBus** with a `JobHandler`
registry. Every background LLM task now rides one seam — the diary's
`session`/`summary` jobs today, procedural directive extraction and future jobs
(semantic dedup, fact merge) next. This is the foundation for a unified aux-LLM
architecture (0.5.0 seam → 0.5.1 observability → 0.6.0 procedural-via-aux).

Nexus stays 100% local-first: the bus only *enqueues* jobs (a local INSERT) and
dispatches host-supplied results to handlers — it never holds an LLM client and
never calls a model.

## Added

- **`core/auxbus/` — the shared AuxBus + `JobHandler` ABC.** `AuxBus` owns the job
  outbox (the table, kept named `summarization_jobs` for zero-DDL compatibility),
  a `{kind: JobHandler}` registry, and the drain/submit dispatch. A handler
  implements `parse_result` (defensive — never raises) and `apply` (commits under
  the shared write lock); a new background job type is just a new registered
  handler, with no orchestrator, table, or host-plumbing change.
- **Unified host API: `drain_aux(run_job, kind=None, limit=None)`,
  `pending_aux_jobs(kind=None, limit=None)`, `submit_aux_job(job_id, result)`.**
  One drain loop across all job kinds; `submit_aux_job` dispatches by `kind`
  (an unknown kind is skipped with a warning, never a `KeyError`). At 0.5.0 the
  bus is **diary-scoped** (constructed only when the diary is enabled); a
  non-diary database is byte-identical to before (no `summarization_jobs` table).

## Changed

- **The diary (Layer V) now rides the AuxBus.** Its job-table SQL moved out of
  `DiaryStore` (which keeps only the two narrative tables, `diary_sessions` /
  `persistent_summary`) into the bus; `_apply_session` / `_apply_summary` became
  `DiarySessionHandler` / `DiarySummaryHandler` registered on the bus. The diary
  produces byte-identical sessions, summaries and folds. `drain_diary` /
  `pending_summaries` / `submit_summary` and the handoff job shape are unchanged
  (pinned to the diary kinds), so existing hosts need no changes.

## Compatibility

- No database migration. No public API removed. `diary=True` / `drain_diary`
  behave exactly as before. The new aux API is additive.

## Verification

- Full suite green (**223 passed**, including the diary lifecycle tests and a new
  `tests/test_aux_bus.py` covering unknown-kind safety, the disabled-bus error
  shape, and the diary riding the bus). Verified end-to-end in the demo: the
  offline self-test plus a real diary run on the aux model produced a correct
  session summary entirely through the new bus path.
