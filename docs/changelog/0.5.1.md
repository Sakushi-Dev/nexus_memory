# 0.5.1 — 2026-06-21

Observability + routing for the [AuxBus](../design/unified-aux-bus.md) introduced
in 0.5.0. Purely additive — no behavior change, no schema change, no public API
removed.

## Added

- **`inspect(type="aux")`** — a read-only snapshot of the background-job outbox:
  `{pending, by_kind, oldest, aux_connected, kinds_registered}`. `aux_connected`
  flips to `True` once any job has completed, so an "aux enabled but never
  drained" host is **visible** instead of silently doing nothing. Returns the
  `aux bus not enabled` error when the bus is off (diary-scoped at 0.5.x).
- **Per-kind routing in `drain_aux` / `AuxBus.drain`.** `run_job` may now be a
  `{kind: callable}` mapping (with an optional `"default"` key) in addition to a
  single callable — so a host can route a strict-JSON kind to a JSON-reliable
  model without a second outbox or a second drain loop. A job whose kind has no
  callable is skipped and left pending (a WARNING, never a raise).

## Verification

- Full suite green (**226 passed**): 3 new tests cover the `inspect(type="aux")`
  snapshot (incl. the `aux_connected` flip after a drain), per-kind routing
  (miss → skipped + left pending; hit → applied), and the `"default"` route.
  Demo offline self-test still passes end-to-end.
