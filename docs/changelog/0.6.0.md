# 0.6.0 — 2026-06-21

Procedural directive extraction moves from an inline regex to the **aux LLM**, via
a new `procedural_extract` job on the shared AuxBus. This is the payoff of the
unified-aux-bus rollout: behavioral rules are now mined with real language
understanding (Mem0-style `ADD` / `UPDATE` / `DELETE` / `NOOP`), and reply language
is excluded **semantically** — the model tells a reply-language wish ("antworte auf
Deutsch") apart from a genuine rule that merely mentions a language ("zitiere immer
deine Quellen"), which the old regex never could.

Zero DDL (the outbox table stays `summarization_jobs`). Nexus still never calls a
model — it enqueues; the host drains via `drain_aux`.

## Added

- **`AuxConfig(enabled=True, procedural_extraction=True)`** and a new
  `NexusMemory(..., aux=...)` parameter (`None`/`True` → enabled, `False` →
  disabled, or a full `AuxConfig`). Procedural-via-aux is **on by default**.
- **`DirectiveExtractHandler`** (`procedural_extract` kind) + a Nexus-owned
  `PROCEDURAL_EXTRACTION_PROMPT`: the model returns a JSON array of ops
  (`ADD`/`UPDATE`/`DELETE`/`NOOP`) over the active directives, with an explicit
  reply-language exclusion. `parse_result` is defensive — malformed/empty/non-JSON
  → all-NOOP (never raises), counted in `parse_failures`.
- **`ProceduralStore.deactivate_by_directive(directive)`** — retract a directive
  by text (powers the handler's `DELETE`).
- **`inspect(type="aux")`** now also reports `parse_failures` and `procedural_via`
  (`"aux"` once a procedural job has completed, else `"regex-fallback"`).

## Changed

- **The aux bus is always-on by default** (no longer diary-scoped): it is built
  whenever `aux.enabled` **or** the diary is on. A plain `NexusMemory()` now has an
  aux bus and enqueues procedural jobs. A host wanting the old "no bus / no
  `summarization_jobs`" shape sets `aux=False` (and no diary).
- **Procedural rules now come from the aux LLM** (`source="aux"`) when aux is on.
  Until the first aux procedural drain completes, an inline-regex **bridge** still
  mines the basic tone/persona rules (`source="auto"`), so a host that has not
  drained yet is never worse off than before. With `aux=False`, procedural is the
  exact 0.4.2 inline-regex path (`source="auto"`, immediate).

## Compatibility

- No database migration; no public API removed. `drain_diary` / `pending_summaries`
  / `submit_summary` and `diary=True` are unchanged. To use procedural-via-aux a
  host drains the bus with `drain_aux(run_job)` (the job's pre-rendered
  `input_text` makes one uniform callable serve every kind); to keep the legacy
  regex behavior, set `aux=False`.

## Verification

- Full suite green (**231 passed**): 5 new tests cover apply-ops, the
  reply-language exclusion (host contract), malformed-JSON → all-NOOP +
  `parse_failures`, the regex-bridge → aux transition, and the `aux=False` inline
  path. Verified end-to-end in the demo on a real aux model (OpenRouter): a
  "reply in German" instruction stored **no** directive, while "keep it short" and
  "always cite your sources" became proper `aux` rules — the original
  language-vs-rule case, finally solved semantically.
